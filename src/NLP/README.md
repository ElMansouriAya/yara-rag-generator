## Overview

This branch contains the complete **Natural Language Processing (NLP) pipeline** used to build a structured cybersecurity knowledge base for the Automatic YARA Rule Generation project.

The objective of the NLP component is to transform heterogeneous cybersecurity resources into a clean, standardized dataset that can later be indexed and queried by the Retrieval-Augmented Generation (RAG) system.

The final output produced by this branch is:

```
data/processed/filtered/dataset_production_enriched.json
```

This dataset serves as the input for the RAG module.

---

# Main Responsibilities

The NLP pipeline is responsible for:

* Loading cybersecurity documents from multiple formats.
* Cleaning and normalizing extracted text.
* Extracting malware-related entities and indicators.
* Building a unified JSON schema.
* Generating optimized text for semantic embeddings.
* Applying quality filtering and validation.
* Enriching the dataset with synthetic cybersecurity records.

# Final Dataset

The final dataset contains **3,046 structured cybersecurity records** derived from malware reports, YARA rules, threat intelligence documents, and synthetic enrichment.

Each record follows a standardized schema including:

* `id`
* `description`
* `malware_family`
* `malware_type`
* `behavior_summary`
* `attack_stage`
* `ioc`
* `yara_rule`
* `rule_conditions`
* `strings_or_patterns`
* `source_document`
* `confidence`
* `notes_on_fp`
* `language`
* `embedding_text`
* `source_type`

---

# Embedding Text

The field `embedding_text` is specifically designed for semantic vectorization.

It combines the most informative attributes of each record, including:

* malware description
* malware family
* malware type
* behavioral characteristics
* attack stage
* relevant indicators of compromise (IOCs)
* extracted patterns and keywords

This field is intended to be used directly by downstream retrieval systems.

---

# Synthetic Data Augmentation

To improve dataset diversity and retrieval performance, synthetic records are generated using several strategies:

* Semantic variations of existing samples
* Cross-family malware scenarios
* MITRE ATT&CK–inspired behaviors
* Templates for well-known malware families

Synthetic records can be identified using:

* IDs beginning with `SYN-`
* `source_type = "synthetic"`

# Running the Pipeline

Run the complete preprocessing workflow:

```bash
python run_preprocessing.py
```

Run quality filtering:

```bash
python src/validation/data_quality_filter.py
```

Generate synthetic records:

```bash
python src/augmentation/synthetic_data_generator.py
```

# Role within the Global Project

The NLP component prepares and structures cybersecurity knowledge. It does **not** perform vector indexing or YARA rule generation. Instead, it provides a high-quality knowledge base that downstream RAG and LLM modules can use for semantic retrieval and automated YARA rule generation.
